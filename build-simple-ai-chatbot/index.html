server/main.py FastAPI backend — the brain of the operation 
"""
main.py  –  FastAPI backend for the AI Chatbot
------------------------------------------------
Run it with:
    uvicorn main:app --reload --port 8000
"""

import os
import uuid
from dotenv import load_dotenv
import anthropic

from fastapi import FastAPI, HTTPException, Header
from fastapi.middleware.cors import CORSMiddleware
from pydantic import BaseModel

# ── 1. Load .env ─────────────────────────────────────────────────────────────
load_dotenv()

# ── 2. Constants ──────────────────────────────────────────────────────────────
CLAUDE_MODEL  = "claude-3-5-sonnet-20241022"
SYSTEM_PROMPT = "You are a helpful, friendly assistant."
MAX_TOKENS    = 1024

# ── 3. In-memory session store ────────────────────────────────────────────────
# sessions[session_id] = [{"role":"user","content":"..."}, ...]
sessions: dict[str, list[dict]] = {}

# ── 4. Anthropic client ───────────────────────────────────────────────────────
api_key = os.getenv("ANTHROPIC_API_KEY")
if not api_key:
    raise RuntimeError("ANTHROPIC_API_KEY not set. Check server/.env")

claude = anthropic.Anthropic(api_key=api_key)

# ── 5. FastAPI app ────────────────────────────────────────────────────────────
app = FastAPI(title="AI Chatbot Backend")

app.add_middleware(
    CORSMiddleware,
    allow_origins=["*"],   # ← Change to your frontend URL in production
    allow_methods=["*"],
    allow_headers=["*"],
)

# ── 6. Schemas ────────────────────────────────────────────────────────────────
class ChatRequest(BaseModel):
    message: str

class ChatResponse(BaseModel):
    reply: str

# ── 7. Chat endpoint ──────────────────────────────────────────────────────────
@app.post("/api/chat", response_model=ChatResponse)
async def chat(
    body: ChatRequest,
    x_session_id: str | None = Header(default=None),
):
    message_text = body.message.strip()
    if not message_text:
        raise HTTPException(status_code=400, detail="Message cannot be empty.")

    session_id = x_session_id or str(uuid.uuid4())

    if session_id not in sessions:
        sessions[session_id] = []

    history = sessions[session_id]
    history.append({"role": "user", "content": message_text})

    try:
        response = claude.messages.create(
            model=CLAUDE_MODEL,
            max_tokens=MAX_TOKENS,
            system=SYSTEM_PROMPT,
            messages=history,
        )
    except anthropic.AuthenticationError:
        raise HTTPException(status_code=500, detail="Invalid Anthropic API key.")
    except anthropic.RateLimitError:
        raise HTTPException(status_code=429, detail="Rate limit reached. Try again.")
    except anthropic.APIStatusError as e:
        raise HTTPException(status_code=502, detail=f"Anthropic error: {e.message}")
    except Exception as e:
        raise HTTPException(status_code=500, detail=f"Unexpected error: {str(e)}")

    reply_text = response.content[0].text
    history.append({"role": "assistant", "content": reply_text})

    return ChatResponse(reply=reply_text)

# ── 8. Health check ───────────────────────────────────────────────────────────
@app.get("/health")
async def health():
    return {"status": "ok"}

client/app.js All frontend JavaScript 
/**
 * app.js – Frontend logic for the AI Chatbot
 * Change API_BASE_URL when you deploy the backend!
 */

// ── 1. Config ──────────────────────────────────────────────────────────────
const API_BASE_URL = "http://localhost:8000"; // ← change for production

// ── 2. Session ID (stored in localStorage) ────────────────────────────────
function getOrCreateSessionId() {
  let id = localStorage.getItem("chatSessionId");
  if (!id) {
    id = crypto.randomUUID();
    localStorage.setItem("chatSessionId", id);
  }
  return id;
}
const SESSION_ID = getOrCreateSessionId();

// ── 3. DOM refs ───────────────────────────────────────────────────────────
const transcript   = document.getElementById("transcript");
const messageInput = document.getElementById("message-input");
const sendBtn      = document.getElementById("send-btn");
const errorBanner  = document.getElementById("error-banner");
const errorText    = document.getElementById("error-text");
const charCount    = document.getElementById("char-count");
const welcomeEl    = document.getElementById("welcome");
const MAX_CHARS    = 2000;

// ── 4. Auto-resize textarea ───────────────────────────────────────────────
messageInput.addEventListener("input", () => {
  messageInput.style.height = "auto";
  messageInput.style.height = Math.min(messageInput.scrollHeight, 140) + "px";
  updateCharCount();
});

function updateCharCount() {
  const len = messageInput.value.length;
  charCount.textContent = `${len} / ${MAX_CHARS}`;
  charCount.className = "char-count";
  if (len > MAX_CHARS * 0.85) charCount.classList.add("warn");
  if (len > MAX_CHARS)        charCount.classList.add("over");
}

// ── 5. Send on Enter (Shift+Enter = newline) ──────────────────────────────
messageInput.addEventListener("keydown", (e) => {
  if (e.key === "Enter" && !e.shiftKey) {
    e.preventDefault();
    handleSend();
  }
});
sendBtn.addEventListener("click", handleSend);

// ── 6. Clear button ───────────────────────────────────────────────────────
document.getElementById("clear-btn").addEventListener("click", () => {
  const newId = crypto.randomUUID();
  localStorage.setItem("chatSessionId", newId);
  location.reload();
});

// ── 7. Main send handler ──────────────────────────────────────────────────
async function handleSend() {
  const text = messageInput.value.trim();

  if (!text) { showError("Please type a message."); return; }
  if (text.length > MAX_CHARS) {
    showError(`Too long (max ${MAX_CHARS} chars).`); return;
  }

  hideError();
  hideWelcome();
  appendMessage("user", text);
  messageInput.value = "";
  messageInput.style.height = "auto";
  updateCharCount();
  setLoading(true);

  try {
    const response = await fetch(`${API_BASE_URL}/api/chat`, {
      method: "POST",
      headers: {
        "Content-Type": "application/json",
        "X-Session-Id": SESSION_ID,
      },
      body: JSON.stringify({ message: text }),
    });

    if (!response.ok) {
      let detail = `Server error (${response.status})`;
      try { const e = await response.json(); detail = e.detail || detail; } catch (_) {}
      throw new Error(detail);
    }

    const data = await response.json();
    appendMessage("ai", data.reply);

  } catch (err) {
    showError(err.message || "Could not reach the server.");
  } finally {
    setLoading(false);
    messageInput.focus();
  }
}

// ── 8. Render a message bubble ────────────────────────────────────────────
function appendMessage(role, text) {
  const wrapper = document.createElement("div");
  wrapper.classList.add("message", role);

  const avatar = document.createElement("div");
  avatar.classList.add("avatar");
  avatar.textContent = role === "user" ? "🧑" : "🤖";

  const bubble = document.createElement("div");
  bubble.classList.add("bubble");
  bubble.textContent = text;   // textContent prevents XSS

  wrapper.appendChild(avatar);
  wrapper.appendChild(bubble);
  transcript.appendChild(wrapper);
  scrollToBottom();
}

// ── 9. Loading dots ───────────────────────────────────────────────────────
let loadingRow = null;
function setLoading(isLoading) {
  sendBtn.disabled = isLoading;
  messageInput.disabled = isLoading;
  if (isLoading) {
    loadingRow = document.createElement("div");
    loadingRow.classList.add("loading-row");
    loadingRow.innerHTML = `
      <div class="avatar">🤖</div>
      <div class="loading-dots">
        <span></span><span></span><span></span>
      </div>`;
    transcript.appendChild(loadingRow);
    scrollToBottom();
  } else {
    if (loadingRow) { loadingRow.remove(); loadingRow = null; }
  }
}

// ── 10. Error helpers ─────────────────────────────────────────────────────
function showError(msg) {
  errorText.textContent = msg;
  errorBanner.classList.remove("hidden");
}
function hideError() {
  errorBanner.classList.add("hidden");
  errorText.textContent = "";
}
function hideWelcome() { if (welcomeEl) welcomeEl.style.display = "none"; }
function scrollToBottom() { transcript.scrollTop = transcript.scrollHeight; }

messageInput.addEventListener("input", hideError);

🔒 CORS Configuration

What is CORS and why does it matter?

CORS (Cross-Origin Resource Sharing) is a browser security feature that blocks a web page from calling an API on a different domain/port unless the server explicitly allows it. Our frontend (port 3000) and backend (port 8000) are different origins, so we must configure CORS.

In server/main.py we add the CORSMiddleware with allow_origins=["*"] (allow all origins). In production, replace "*" with your exact Netlify URL like "https://my-chatbot.netlify.app" for better security. 

# In server/main.py – change for production:
app.add_middleware(
    CORSMiddleware,
    allow_origins=["https://my-chatbot.netlify.app"],  # ← your Netlify URL
    allow_methods=["POST", "GET"],
    allow_headers=["Content-Type", "X-Session-Id"],
)

🛠️ Common Problems & Fixes

❌ ANTHROPIC_API_KEY is not set — RuntimeError on startup

Cause: You haven't created server/.env, or it's in the wrong directory, or the variable name is misspelled.

Fix:

  1. Make sure the file is literally named .env (not .env.txt)
  2. It must be inside the server/ folder (same directory as main.py)
  3. The variable name must be exactly ANTHROPIC_API_KEY (all caps, no spaces)
  4. Run uvicorn from inside the server/ directory
❌ CORS error in browser console

Cause: The browser is blocking the request because the origin doesn't match what the server allows.

Fix (local):

  1. Make sure FastAPI is running (uvicorn on port 8000)
  2. Make sure API_BASE_URL in app.js is http://localhost:8000
  3. Try serving the client with python3 -m http.server 3000 instead of opening the file directly

Fix (production): Replace allow_origins=["*"] in main.py with your exact Netlify URL, redeploy the backend.

❌ "Could not reach the server" — fetch fails

Cause: The FastAPI server isn't running, or the URL is wrong.

Fix:

  1. Confirm uvicorn is running in your terminal (you should see Uvicorn running on http://127.0.0.1:8000)

❌ Invalid Anthropic API key — 401 / AuthenticationError

Cause: The key in .env is wrong, expired, or has extra spaces.

Fix:

  1. Open server/.env and check there are no quotes around the key: ANTHROPIC_API_KEY=sk-ant-… (no quotes)
  2. No spaces around the = sign
  3. Generate a fresh key at console.anthropic.com if needed
  4. Restart uvicorn after changing .env
❌ Rate limit error — 429

Cause: You've sent too many requests in a short time (common with free Anthropic accounts).

Fix: Wait 60 seconds and try again. If it happens often, add credit to your Anthropic account or reduce MAX_TOKENS in main.py.

❌ Render app sleeps after 15 minutes (free tier)

Cause: Render's free Web Services spin down after inactivity. The first request after sleep takes ~30 seconds.

Fix options:

  1. Accept the cold start (it's free!)
  2. Use a cron job service (e.g., cron-job.org) to ping your /health endpoint every 14 minutes
  3. Upgrade to Render's paid tier ($7/month) for always-on
❌ Windows: venv\Scripts\activate gives an error

Cause: PowerShell execution policy blocks scripts.

Fix: Run this in PowerShell as Administrator, then try again:

Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser

Or use Command Prompt (cmd.exe) instead of PowerShell.
